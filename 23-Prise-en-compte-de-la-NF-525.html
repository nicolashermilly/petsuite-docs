Obligation légale — Art. 88 LF2016

Tout logiciel encaissant des paiements de clients particuliers assujettis à la TVA doit satisfaire aux conditions d'inaltérabilité, sécurisation, conservation et archivage de la norme NF 525. Amende : 7 500 € par logiciel non conforme, renouvelable.

Dépendances
BLOQUANTScript_anim_gest_v3.sql
Le script NF525 doit être exécuté après le script métier v3. Toutes les FK vers facture, reglement, avoir, tiers et utilisateur doivent exister.
BLOQUANTDroits SQL — @lgorithme
L'utilisateur applicatif doit avoir EXECUTE sur les 4 procédures NF525, INSERT sur les tables du journal. Pas de DROP/ALTER/TRUNCATE.
INFOMariaDB 10.6+
Requis pour DATETIME(3), JSON natif et SHA2() en triggers. Vérifier la version de l'hébergeur avant déploiement.
INFOGED v3 (Sprint S31)
Les tickets Z imprimés peuvent être archivés via la GED (type objet CLOTURE_Z). À connecter en S32 si besoin.

ℹ️
Point d'attention critique — Intégration API

Le script SQL seul ne garantit pas la conformité NF525. L'appel à sp_nf525_enregistrer_encaissement doit être déclenché depuis la couche applicative, dans la même transaction que l'INSERT dans reglement. C'est le point bloquant de l'US-23.2.
