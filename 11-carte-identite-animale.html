01Concept & principes fondateurs
🐾
La puce comme clé universelle
Format15 chiffres — standard ISO 11784/11785
Unicité mondialeIdentifiant implanté, non modifiable, unique

Cross-tenantLa puce permet de retrouver l'animal dans n'importe quelle société PetSuite

04Architecture technique
Génération PDF — choix technique
🖨
Option A — Puppeteer (recommandé)
PrincipeRendu HTML → capture PDF via Chromium headless
AvantageMise en page identique à la maquette HTML — CSS complet
PhotosImages GED intégrées nativement via base64
IntégrationModule NestJS dédié — ou micro-service isolé
InconvénientChromium lourd (~300 Mo) — à isoler dans conteneur dédié
Recommandénpm install puppeteer
📝
Option B — pdfkit
PrincipeGénération programmatique du PDF (drawText, drawImage…)
AvantageLéger — pas de Chromium — déjà utilisable dans micro_api
PhotosIntégration images possible mais moins flexible
InconvénientMise en page manuelle — plus laborieuse à maintenir
Alternative légèrenpm install pdfkit
Stratégie de stockage
Pas de stockage permanentLe PDF est généré à la demande et retourné en streaming — pas sauvegardé sur disque
Exception : partageSi l'utilisateur génère un lien de partage, le PDF est temporairement stocké dans le bucket S3 GED du tenant avec TTL 15 min
Chemin S3 temporaire/{tenant_id}/cartes-tmp/{uuid-generation}.pdf — suppression auto après TTL
Cache optionnelRedis — cache le PDF 5 min pour éviter les régénérations multiples sur la même fiche
Invalidation cacheToute modification de l'animal ou d'une séance invalide le cache immédiatement
Endpoints API
MéthodeRouteActionRetour
GET/api/animaux/:id/carte-pdfGénère et retourne le PDF en streamingapplication/pdf

GET/api/animaux/:id/carte-configRetourne les options de génération disponibles{ sections, options }

Body de génération personnalisée
POST /api/animaux/42/carte-pdf { "sections": { "identite": true, // toujours true "cac": true, // grille 14 critères "progression_cac": true, // tableau évolution score "seances": true, // historique séances "comptes_rendus": false, // textes CR (optionnel) "photos": true, // photos GED "documents_ged": false // liens docs attachés }, "nb_seances_max": 10, // limiter l'historique "inclure_proprietaire_tel": false // masquer tél. si partage }
05Sécurité & RGPD
URL signée éphémère
🔐
Fonctionnement
Durée de vie15 minutes par défaut — paramétrable jusqu'à 24h
Structure tokenJWT signé HMAC-SHA256 : ani_id + tenant_id + sections + exp
AccèsRoute publique /api/carte/:token — pas d'authentification requise
Usage uniqueOption : token invalidé après premier accès
TraçabilitéChaque accès loggé : IP, date/heure, tenant_id source
🔗
Exemple token
// Payload JWT { "ani_id": 42, "tenant_id": "tenant_0042", "sections": ["identite", "cac"], "masquer_tel": true, "iat": 1710410400, "exp": 1710411300 // +15 min } // URL partagée https://app.petsuite.fr/api/carte/ eyJhbGciOiJIUzI1NiJ9...
Consentement propriétaire (RGPD)
1
Déclenchement partage
L'utilisateur clique "Générer un lien de partage" depuis la fiche animal. Une modale apparaît avant toute génération.
2
Confirmation consentement
Modale : "Confirmez-vous que le propriétaire [Famille Rousseau] a donné son accord pour partager ces informations ?" — bouton Confirmer obligatoire.
3
Enregistrement traçabilité
INSERT dans table partage_carte : ani_id, tenant_id, utilisateur, date, destinataire (optionnel), token, sections partagées. Conservé 5 ans.
4
Génération URL + copie
URL copiée dans le presse-papier. Bouton "Envoyer par email" disponible. Validité affichée clairement (ex: "expire dans 15 min").
Le partage crée une traçabilité RGPD obligatoire. Chaque partage doit être justifiable par le consentement du propriétaire de l'animal. Les données personnelles du propriétaire (téléphone, email) ne sont jamais incluses par défaut dans un partage.
06Partage cross-tenant (vision SaaS)
Fonctionnalité avancée — à implémenter après la mise en production mono-tenant. La puce comme clé universelle permet de retrouver un animal entre sociétés PetSuite distinctes.
🌐
Scénario type
A
Société A partage la carte
Cabinet Canin Harmonie génère un lien de partage pour Titan (puce 250269811500001) et l'envoie au vétérinaire du Dr Dupont.
B
Accès via URL signée
Le vétérinaire (Société B — autre tenant PetSuite) accède au PDF via le lien. Lecture seule des données autorisées uniquement.
C
Recherche par puce (optionnel)
Si le vétérinaire est client PetSuite et saisit la puce dans son interface, il peut demander accès à la carte via un workflow de consentement.
⚠️
Contraintes strictes
Pas d'accès direct BDDJamais d'accès au schema d'un autre tenant — uniquement via le token signé
Données minimalesSeules les sections explicitement autorisées sont exposées
Données financièresToujours exclues — jamais partagées entre tenants
Consentement obligatoireChaque partage nécessite confirmation explicite
Audit completChaque accès cross-tenant loggé dans audit_plateforme
RévocationLe propriétaire ou l'admin société peut révoquer tout partage actif
