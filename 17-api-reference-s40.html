Architecture API

NestJS + custom-routes.ts

Routes NestJS (TypeORM)

Controllers générés automatiquement. Modules : TiersModule, AnimauxModule, DevisModule, FactureModule, ComptaModule, EditionsModule...

Toutes les routes NestJS sont mappées dans app.module.ts.

custom-routes.ts

Routes Express déclarées avec app.use() AVANT app.listen() dans main.ts. Utilise mysql2/promise directement (pas TypeORM).

Connexion : env vars DB_HOST, DB_PORT, DB_USER, DB_PASSWORD, DB_NAME.

⚠️ Règle nginx : nginx strip le préfixe /api/ avant proxy vers l'API. Les routes internes se déclarent SANS /api/ (ex: '/reglements' pas '/api/reglements').
💡 Patch routes : utiliser des scripts Python (pas PowerShell) pour patcher custom-routes.ts — évite les problèmes d'encodage avec les backticks TypeScript.
