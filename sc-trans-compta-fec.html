01 Étapes du scénario
01 Configuration du plan comptable
  • Comptabilité → Plan comptable : adapté au métier choisi
  • Comptes obligatoires : 411 (clients), 401 (fournisseurs), 706 (prestations), 707 (ventes)
  • Comptes TVA : 44566 (déductible), 44571 (collectée), 44551 (à payer)
  • Comptes spécifiques métier : 6064 (consommables), 6257 (repas), 6251 (déplacements)
  • Personnalisation possible : ajouter des sous-comptes (706.1 toilettage, 706.2 pension...)
02 Création de l'exercice comptable
  • Comptabilité → Exercices → + Nouvel exercice
  • Date début : 01/01/2026, date fin : 31/12/2026
  • Statut : Ouvert (modifiable) → Clôturé (verrouillé)
  • Un seul exercice actif à la fois pour les nouvelles écritures
03 Écriture automatique — Vente d'une prestation
Lors de la facturation, l'écriture est générée automatiquement.
  • Facture 120€ TTC = 100€ HT + 20€ TVA 20%
  • Journal AC (ventes) :
  •   • Débit 411 (Client X) : 120,00 €
  •   • Crédit 706 (Prestations) : 100,00 €
  •   • Crédit 44571 (TVA collectée) : 20,00 €
  • Libéllé : FAC-2606-0078 / Client X / BC Rex
04 Écriture automatique — Règlement client
  • Lors de l'encaissement Stripe/virement : nouvelle écriture
  • Journal BQ (banque) :
  •   • Débit 512 (Banque) : 120,00 €
  •   • Crédit 411 (Client X) : 120,00 €
  • Le compte client est soldé automatiquement
  • Rapprochement bancaire facilité (même référence dans les deux écritures)
05 Consultation de la balance
  • Comptabilité → Balance — vue par compte
  • Pour chaque compte : Solde début, Mouvements débit/crédit, Solde fin
  • Filtres : période, classe de compte, journal
  • Compte 411 (clients) : solde débiteur = factures impayées
  • Compte 706 (ventes) : solde créditeur = CA du mois
  • Export PDF ou CSV de la balance
06 Génération du FEC
  • Comptabilité → Export FEC
  • Format normalisé : 18 colonnes (article L47 A-I du LPF)
  • JournalCode, JournalLib, EcritureNum, EcritureDate, CompteNum, CompteLib, CompAuxNum, CompAuxLib, PieceRef, PieceDate, EcritureLib, Debit, Credit, EcritureLet, DateLet, ValidDate, Montantdevise, Idevise
  • Format texte tabulé, UTF-8, conforme à l'arrêté du 29 juillet 2013
  • Nom de fichier : SIREN_FECYYYYMMDD.txt
07 Vérification qualité du FEC
  • Avant export : check-list automatique de conformité
  • Toutes les écritures sont équilibrées (somme débits = somme crédits)
  • Aucune écriture en doublon (même référence)
  • Tous les comptes utilisés existent dans le plan comptable
  • Format des dates et montants conforme
  • Test validation Test-Compta-Demat (DGFiP) recommandé avant transmission
✓ Bonne pratique : Un FEC propre évite les demandes de correction lors d'un contrôle fiscal.
08 Transmission et clôture de l'exercice
  • Transmission FEC à l'expert-comptable par email ou via plateforme sécurisée
  • L'expert-comptable réalise les écritures d'inventaire (amortissements, provisions)
  • Renvoi du FEC final pour archivage
  • Comptabilité → Exercices → Clôturer : l'exercice est verrouillé
  • Génération du bilan et du compte de résultat
  • Archive légale 10 ans — conservation dans GED + sauvegarde NAS
09 Export TVA par taux
  • Comptabilité → Export TVA — ventilation par taux
  • Base HT et TVA pour 0%, 5,5%, 10%, 20% séparément
  • Vue mensuelle ou trimestrielle selon votre régime
  • Pré-remplissage des cases CA3 (chiffre d'affaires réalisé HT)
  • Télédéclaration via votre espace impots.gouv.fr
10 Notes de frais et déductions
  • Les notes de frais validées génèrent des écritures dans le journal OD/AC
  • TVA déductible automatiquement enregistrée (44566)
  • Catégories PCG : 6064 (consommables), 6257 (repas), 6251 (déplacements)
  • Export CSV par catégorie pour le bilan annuel
  • Justificatifs scannés (OCR IA) archivés en GED
