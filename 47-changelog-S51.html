🗺️ Carte clients — Leaflet + géocodage
FIX Carte clients — Leaflet ne s'initialisait pas
Le composant VueCarte bloquait l'affichage du <div ref={mapRef}> pendant tout le géocodage batch (loading = true). Résultat : mapRef.current === null au moment où le useEffect Leaflet s'exécutait → la carte n'était jamais montée.
Fix : setDataLoading(false) dès réception des données API, avant le géocodage. La carte s'affiche immédiatement vide, les marqueurs apparaissent progressivement au fur et à mesure du géocodage en arrière-plan.
FIX Carte clients — marqueurs absents (latitude/longitude null en BDD)
La route /api/tiers-avec-coords retourne latitude: null, longitude: null pour tous les clients car TIE_VILLE_ID est null (jointure morte avec la table ville).
Fix : géocodage batch côté frontend via api-adresse.data.gouv.fr en utilisant TIE_CP. Cache par CP pour éviter les appels dupliqués — 149 clients avec CP géocodés en ~50 appels uniques.
Résultat : 140 clients affichés sur la carte sur 156 total (16 sans adresse géocodable).
FIX Carte clients — CDN Leaflet migré unpkg → cdnjs
Le code source utilisait encore unpkg.com/leaflet@1.9.4 au lieu de cdnjs.cloudflare.com/ajax/libs/leaflet/1.9.4/leaflet.min.js (règle F5). Corrigé dans src/app/app/clients/page.tsx.
NEW Carte clients — indicateur de géocodage en cours
Ajout d'un bandeau ⏳ Géocodage en cours... N adresses traitées pendant le batch. Compteur geocodedCount mis à jour en temps réel. Le compteur disparaît automatiquement une fois le géocodage terminé.

🔧 Règles techniques confirmées S51
RULE Leaflet en Next.js 15 — timing useEffect critique
Le div ref={mapRef} doit être présent dans le DOM avant que le useEffect Leaflet s'exécute. Si un état loading conditionne le rendu du div, Leaflet ne peut pas s'initialiser.
Pattern correct : séparer dataLoading (masque tout) de geocoding (s'exécute en arrière-plan sans masquer la carte).
RULE Fichiers [id] sous Windows/OneDrive — lecture via System.IO
Les fichiers src/app/app/xxx/[id]/page.tsx ne sont pas lisibles avec Get-Content (crochets filtrés par PowerShell/OneDrive). Toujours utiliser [System.IO.File]::ReadAllText($path) avec Join-Path $base "xxx\[id]\page.tsx".
TECH Mode clair — --surface-input correctement défini
Audit CSS confirmé : --surface-input: #FFFFFF défini dans le sélecteur racine (thème clair par défaut) et --surface-input: #1E293B dans html.dark. La classe .input utilise var(--surface-input). Pas de bug — tâche S50 était une fausse alerte.
