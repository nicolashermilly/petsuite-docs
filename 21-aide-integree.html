Architecture des composants

Bibliotheque de donnees pure TypeScript. 58 articles, 6 modules, 3 fonctions utilitaires. Aucune dependance externe. Source unique de verite.

Composant client React. Bouton FAB fixe bas-droit + panneau lateral contextuel. Monte dans layout.tsx, actif sur toutes les pages /app/*.

Flux utilisateur

1
L'utilisateur navigue sur n'importe quelle page /app/* de Anim'Gest
2
Le bouton ? fixe bas-droit est toujours visible — clic pour ouvrir le panneau
3
Le panneau detecte la page via usePathname() et affiche 4-5 questions contextuelles
4a
L'utilisateur clique sur une question → accordeon s'ouvre → reponse inline
4b
L'utilisateur tape dans la barre de recherche → resultats filtres en temps reel (2+ caracteres)
5
Clic sur "Toute l'aide" → ferme le panneau → navigue vers /app/aide
6
Centre d'aide : navigation par module, recherche globale, lien direct par ancre URL

Interface TypeScript

export interface HelpArticle {
  id: string;              // ex: "ani-01", "sea-03", "crm-07"
  moduleId: string;         // ex: "animaux", "seances", "crm"
  question: string;         // Question affichee dans l'accordeon
  answer: string;           // Supporte **gras** (rendu en vert)
  pages?: string[];         // Ex: ["/app/animaux/[id]"] — [id] = regex
  tags?: string[];          // Mots-cles pour la recherche (score x2)
}

export interface HelpModule {
  id: string;
  label: string;
  icon: string;              // Emoji
  description: string;
}

Algorithme de recherche (searchArticles)

Zone de correspondanceScore par termeRaison
Question+3Correspondance la plus pertinente
Tags+2Mots-cles curates par le redacteur
Reponse+1Contexte supplementaire

Les scores sont cumules sur tous les termes de la requete. Les articles avec score = 0 sont filtres. Le resultat est trie par score decroissant.

Ajouter ou modifier un article

1
Ouvrir src/lib/help-content.ts
2
Ajouter un objet dans le tableau HELP_ARTICLES (ou modifier un existant)
3
Renseigner id unique, moduleId, question, answer, pages[], tags[]
4
Rebuild : docker compose build frontend && docker compose up -d frontend && docker compose restart nginx
Les routes dynamiques s'ecrivent avec [id] dans le tableau pages. La fonction getArticlesForPage() les convertit automatiquement en regex. Ex: "/app/animaux/[id]" matche /app/animaux/42 et /app/animaux/abc123.
