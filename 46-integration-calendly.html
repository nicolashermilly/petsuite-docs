🗓️ Intégration Calendly

Architecture

⚙️ Configuration
Page admin pour saisir les URLs Calendly par type de RDV. Stockage dans param_config (groupe CALENDLY). Accessible via /app/parametres/calendly.
🌐 Page publique
Page /reservation accessible sans connexion. Grille de sélection des types configurés, puis widget Calendly inline. Aucune auth Google requise.
🗃️ Stockage BDD
Table param_config générique. Clé GLOBAL_URL pour l'URL principale, clés par type de RDV (EC_1H30, SUIVI_1H...) pour les URLs spécifiques.

Flux de réservation client

Étape 1
Client reçoit le lien
URL partagée : https://Anim'Gest.fr/reservation ou /reservation?type=EC_1H30. QR code générable depuis le dashboard.
Étape 2
Sélection du type de RDV
Sans paramètre type → grille des types disponibles avec description et durée. Avec type → passage direct au widget.
Étape 3
Widget Calendly inline
Widget Calendly chargé en iframe avec couleur primaire du métier. Client choisit créneau et remplit ses coordonnées. Confirmation par email automatique.
Étape 4 — S43
Webhook → Séance Anim'Gest
À venir : webhook Calendly → création automatique de séance dans Anim'Gest. Nécessite route POST /webhooks/calendly.

Configuration technique

Table param_config

CREATE TABLE IF NOT EXISTS param_config (
  PC_ID    INT AUTO_INCREMENT PRIMARY KEY,
  PC_GROUP VARCHAR(50)  NOT NULL,  -- ex: 'CALENDLY', 'SMTP', 'SMS'
  PC_KEY   VARCHAR(100) NOT NULL,  -- ex: 'GLOBAL_URL', 'EC_1H30'
  PC_VALUE TEXT NULL,
  UNIQUE KEY uq_pc (PC_GROUP, PC_KEY)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

Éviter la connexion Google

Dans chaque "Event type" Calendly, décocher l'option "Require invitees to have a Google account" et désactiver la sync Google Calendar si non souhaitée. Le widget fonctionne alors sans auth tierce.
