01 Étapes du scénario
01 Comprendre la RFE et votre calendrier
  • RFE = Réforme Facturation Électronique (article 153 LF 2020)
  • Obligation de réception de factures électroniques : tous les assujettis TVA dès le 01/09/2026
  • Obligation d'émission : grandes entreprises et ETI au 01/09/2026, PME/TPE au 01/09/2027
  • Concerne les transactions B2B domestiques (entre entreprises françaises)
  • Les transactions B2C et internationales relèvent du e-reporting
💡 Conseil : Même si vous êtes une TPE/PME (1er sept 2027), préparez-vous dès 2026 car vos clients grands comptes vous demanderont la conformité.
02 Choisir votre PDP (Plateforme de Dématérialisation Partenaire)
  • Deux options : PPF (Portail Public de Facturation, gratuit) ou PDP (privée, payante mais riche en fonctions)
  • PDP recommandées : Sage, Cegid, Dext, Yooz, Esker, Generix, Invéteris, Pennylane...
  • Critères : compatibilité ERP, prix, fonctionnalités avancées (recouvrement, archivage)
  • Anim'Gest sera compatible avec les principales PDP via API standardisée
  • PPF gratuit suffit pour les flux simples ; PDP utile pour automatisations avancées
03 Générer une facture au format Factur-X
  • Factur-X = PDF/A-3 + fichier XML structuré (norme franco-allemande)
  • Anim'Gest génère déjà les factures au format compatible Factur-X
  • Niveau MINIMUM : données essentielles (TVA, montants)
  • Niveau BASIC WL : ajout des lignes simplifiées
  • Niveau EN 16931 (recommandé) : conformité norme européenne complète
  • Niveau EXTENDED : extensions spécifiques (rare)
04 Cycle de vie d'une facture électronique B2B
  • 1. Dépôt : l'émetteur dépose sur sa PDP (ou PPF)
  • 2. Routage : la PDP transmet à la PDP du destinataire (via annuaire central PPF)
  • 3. Réception : le destinataire reçoit dans sa PDP
  • 4. Statuts obligatoires : Déposée → Reçue → Approuvée (ou Refusée)
  • 5. Paiement : encaissement notifié à la PDP
  • 6. Archivage 10 ans dans la PDP + e-reporting des données vers DGFiP
05 E-reporting — Vos obligations de déclaration
  • E-reporting = transmission des données de transactions à la DGFiP
  • Concerne : B2C, B2B international, transactions en espèces ou cartes
  • Fréquence : selon régime TVA (mensuel/trimestriel)
  • Données transmises : montants HT/TVA par taux, nature, dates
  • Anim'Gest générera automatiquement les fichiers e-reporting via votre PDP
  • Objectif DGFiP : pré-remplir la déclaration TVA
06 Mise en place progressive — Plan d'action
  • S1-S2 2026 : choisir et abonner votre PDP/PPF
  • S2 2026 : configurer l'envoi automatique vers PDP depuis Anim'Gest
  • Avant 01/09/2026 : tester la réception de factures fournisseurs
  • Avant 01/09/2027 (TPE/PME) : généraliser l'émission Factur-X
  • Former l'équipe administrative (1 demi-journée suffit)
  • Mettre à jour les CGV avec les nouvelles obligations
07 Vérification de conformité
  • Vos factures contiennent toutes les 20 mentions obligatoires (existantes + nouvelles)
  • Nouveau : adresse de livraison si différente de l'adresse de facturation
  • Nouveau : option pour le paiement de la TVA sur les débits (si choisi)
  • Nouveau : type d'opération (livraison de biens / prestation de services / mixte)
  • Identifiant SIREN obligatoire du destinataire pour les factures B2B
  • Test : envoyer une facture pilote via votre PDP avant le 01/09/2026
