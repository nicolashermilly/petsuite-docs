Étape 1 Comprendre la RFE et votre calendrier
La Réforme de la Facturation Électronique s'applique progressivement.
  • Septembre 2026 : grandes entreprises (CA > 250M€) — obligation de réception
  • Septembre 2026 : ETI (CA > 50M€) — obligation émission + réception
  • Juillet 2027 : PME et micro-entreprises (VOUS) — obligation émission + réception
  • Votre obligation : émettre des factures au format structuré via une PDP agréée
  • Format obligatoire : Factur-X (PDF+XML) ou UBL ou CII
  • Amende en cas de non-conformité : jusqu'à 15€ par facture manquante
⚠️ Vérifiez votre date exacte selon votre CA. Consultez impots.gouv.fr pour les mises à jour.
2
Étape 2 Choisir votre PDP (Plateforme de Dématérialisation Partenaire)
Vous devez passer par une PDP agréée par l'administration fiscale.
  • Liste des PDP agréées disponible sur impots.gouv.fr
  • Options courantes : Pennylane, Chorus Pro (gratuit B2G), Sellsy, Sage, Cegid
  • Critères de choix : tarif, intégration API, support, volume de factures
  • Anim'Gest → Paramètres → RFE → Configurer votre PDP
  • Renseigner les identifiants API de votre PDP choisie
  • Tester l'envoi d'une facture de test avant la date d'obligation
3
Étape 3 Générer une facture au format Factur-X
Le format Factur-X est un PDF classique avec un fichier XML intégré.
  • Créer votre facture normalement dans Anim'Gest
  • Au moment du PDF : choisir Format Factur-X (au lieu du PDF standard)
  • Le fichier généré contient : PDF lisible + XML structuré EN16931 intégré
  • Le fichier XML contient : SIREN émetteur, SIREN destinataire, lignes de facture, TVA
  • Le client peut l'ouvrir comme un PDF normal ET l'intégrer dans son système comptable
  • Votre PDP lit le XML et transmet la facture au réseau de dématérialisation
💡 Pour les clients particuliers (B2C), le format Factur-X n'est pas obligatoire. PDF standard suffisant.
4
Étape 4 Cycle de vie d'une facture électronique B2B
Une facture électronique passe par plusieurs statuts réglementaires.
  • DEPOSEE : la facture est transmise à votre PDP ✓
  • TRANSMISE : la PDP a transmis au destinataire ✓
  • ACCEPTEE : le destinataire a validé la facture ✓
  • EN_LITIGE : le destinataire conteste (délai de réponse 15 jours)
  • REFUSEE : contestation non résolue → émettre un avoir + nouvelle facture
  • Chaque statut est tracé dans Anim'Gest avec date et heure
  • Un avoir annulant une facture refusée doit aussi passer par la PDP
5
Étape 5 E-reporting — Vos obligations de déclaration
En plus des factures, vous devez déclarer vos transactions B2C.
  • E-reporting : transmission des données de transactions non soumises à facturation électronique
  • Concerne : ventes B2C (particuliers), ventes internationales
  • Fréquence : mensuelle ou trimestrielle selon votre CA
  • Format : fichier structuré transmis à votre PDP → transmis à l'administration
  • Données : montant HT, TVA, devise, type de transaction
  • Anim'Gest génère automatiquement le fichier d'e-reporting depuis vos règlements
6
Étape 6 Mise en place progressive — Plan d'action
Préparez-vous maintenant pour éviter le rush de juillet 2027.
  • Dès maintenant : activer le module RFE dans Paramètres → Modules
  • Choisir votre PDP et configurer les identifiants API (3 mois avant obligation)
  • Former votre équipe : 1 heure suffit pour comprendre le nouveau workflow
  • Tester avec 5 vraies factures clients avant la date d'obligation
  • Informer vos clients B2B : ils devront indiquer leur SIREN sur leurs commandes
  • Mettre à jour vos CGV : mentionner la facturation électronique
7
Étape 7 Vérification de conformité
Utilisez le tableau de bord RFE pour suivre votre conformité.
  • Factures → RFE : tableau de bord des statuts de transmission
  • Indicateurs : % factures transmises, % acceptées, % en litige, % refusées
  • Objectif : 100% acceptées · Alerte si > 2% en litige
  • Rapport mensuel e-reporting : vérifier que toutes les transactions B2C sont déclarées
  • Archive : conserver les accusés de réception PDP pendant 10 ans
  • Contact PDP : numéro de support en cas de problème de transmission
