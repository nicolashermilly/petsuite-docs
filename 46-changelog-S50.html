Routes API — signatures / agenda / planning-séances

FIX
Bug critique dans custom-routes.ts — Les routes /signatures, /agenda et /planning-seances étaient déclarées en dehors de la fonction registerCustomRoutes(app) (après l'accolade fermante à la ligne 1906). Cela provoquait une erreur TypeScript TS2304 au build et les 3 routes retournaient 404.
API
GET /api/signatures — Retourne les signatures Yousign en BDD (table vide, [] normal en dev). GET /api/agenda — Retourne les séances formatées par jour pour la vue agenda. GET /api/planning-seances — Retourne le planning hebdomadaire. Les 3 routes retournent désormais HTTP 200.
API
GET /api/stats — Confirmé inutile (doublon de /api/dashboard/stats qui répond 200 avec {nb_clients, nb_animaux, nb_seances, ca_ht}). Route /api/stats non créée.

Règles techniques confirmées S50

RULE
Routes PDF API : /api/editions/facture/:id et /api/editions/devis/:id — retournent un blob PDF binaire. Ne jamais utiliser /api/factures/:id/pdf (ancienne convention, non implémentée).
RULE
Séquence rebuild frontend : toujours patcher les .tsx avant de lancer COPIER_TOUTES_PAGES_v2.ps1. Si le script est lancé avant le patch, le build embarque l'ancienne version.
RULE
Nginx strip /api/ : les routes custom-routes.ts sont enregistrées sans le préfixe /api/. Nginx retire ce préfixe avant de transmettre au backend.
