🚀 Eliot V2 — En préparation Roadmap S122

Eliot V2 introduira une refonte profonde de l'agent IA, avec quatre apports majeurs : analyse libre de vos données via un nouveau tool query_database, workflow d'action en deux temps avec panneau de préparation et modal de confirmation, audit complet de chaque interaction (qui, quand, quel résultat), et permissions héritées strictes du rôle de l'utilisateur connecté.

Première vague de livraison : métier comportementaliste / éducateur canin (S122, soit ~4 jours de développement). Extension progressive aux 11 autres métiers ensuite, à raison d'un toutes les 1-2 semaines selon la complexité.

Nouveaux outils Eliot V2 S122

Eliot V2 ajoutera 6 nouveaux outils orientés analyse et synthèse métier. La grande nouveauté est query_database qui rendra Eliot capable de répondre à n'importe quelle question analytique.

🛡️
Sécurité query_database : malgré son nom, ce tool ne donne pas d'accès brut à la base. Chaque requête est analysée par un parser SQL côté serveur qui refuse toute opération de modification (INSERT, UPDATE, DELETE, DROP, TRUNCATE) et toute tentative d'accès aux tables sensibles (mots de passe, clés API, audit, signatures électroniques, tokens). Un timeout de 5 secondes empêche les requêtes coûteuses, et un compte technique dédié en lecture seule est utilisé pour l'exécution. Eliot ne peut jamais écrire via ce canal, même si vous le lui demandez explicitement.

Workflow d'action en 2 temps V2

Eliot V2 introduira un workflow systématique pour toute action de modification : Eliot prépare l'action, vous validez. Vous gardez la maîtrise totale, et chaque étape est tracée.

Étape 1
📋
PREPARE
Eliot prépare l'action et affiche un panneau latéral avec le détail complet (entête, lignes, totaux, animal concerné, type séance...). Aucune écriture en base à ce stade.
Étape 2
✋
CONFIRM
Vous cliquez sur « Valider », une modal de confirmation s'ouvre avec récap concis. Vous pouvez annuler ou confirmer. Toutes les étapes sont auditées.
Étape 3
✅
EXECUTE
L'action est exécutée via les API normales du logiciel (même chemin que si vous aviez cliqué dans l'interface). Résultat injecté dans la conversation Eliot.
Étape 4
📊
AUDIT
Chaque étape PREPARE / CONFIRM / EXECUTE est tracée : qui, quand, quoi, durée, statut. Consultable dans Paramètres → Eliot → Historique.
💡
Pourquoi deux étapes UX (panneau puis modal) ? Le panneau donne le contexte complet et le temps de relire (utile pour un devis multi-lignes). La modal impose ensuite une décision binaire claire. L'audit log distingue les deux moments, ce qui permet d'identifier les hésitations et de surveiller les abandons d'action.

Permissions héritées du rôle utilisateur V2

Eliot V2 héritera strictement des droits du compte connecté. Un utilisateur ne pourra demander à Eliot que ce qu'il pourrait faire lui-même dans l'interface.

ADMIN
Tier 3
Tous les outils (lecture, création, modification statut). Configure les niveaux de permission depuis /parametres/eliot-permissions.
MANAGER
Tier 2
Lecture complète + création devis/factures/règlements avec confirmation. Modification statuts commerciaux. Pas d'accès aux paramètres sensibles.
ADMINISTRATIF
Tier 1
Lecture complète + planification de séances + envoi de liens portail client. Pas de création de devis/factures via Eliot (refusé par permission).
🔐
3 niveaux par outil configurables dans Paramètres → Eliot → Permissions :
  • • AUTO — exécution directe (lecture uniquement par défaut)
  • • CONFIRM — passe par le workflow PREPARE → CONFIRM
  • • DENIED — outil désactivé pour ce compte

Commandes complexes — Boucles agentiques

Eliot peut enchaîner jusqu'à 6 actions pour répondre à une demande complexe. Exemples qui illustrent la puissance de l'agent quand il combine plusieurs outils.

Rapport impayés complet — V1 disponible
« Donne-moi la liste de tous les clients avec des impayés, triée par montant décroissant, avec leur téléphone pour que je puisse les rappeler. »

→ Eliot appelle list_factures (filtre impayées) → regroupe par client → get_client_details (pour le téléphone) → formate et présente la liste en 5 secondes.
Préparation semaine complète — V1 disponible
« Prépare ma semaine du 20 au 24 mai : liste les séances, vérifie les impayés des clients concernés et dis-moi si des animaux ont des vaccinations qui expirent ce mois. »

→ 3 appels parallèles : get_agenda + list_factures (filtre clients de la semaine) + search_animals (filtre vaccins mois) → synthèse complète.
Analyse business multi-dimensions V2 — Nouveau
« Analyse mon activité du trimestre : top 5 clients en CA, évolution comparée à T-1, clients perdus depuis 6 mois, et stocks à risque. »

→ query_database avec 4 requêtes SQL générées par Eliot, validées par parser, exécutées sur le compte lecture seule → synthèse formatée. Impossible en V1 (pas d'outil analytique libre).
Création en cascade avec validation V2 — Workflow
« Crée un nouveau client, Jean Martin, jean.martin@gmail.com, 06 12 34 56 78, avec son chien Max (Labrador 3 ans) et planifie un BC pour lundi prochain à 10 h. »

→ Eliot prépare les 3 actions et les présente dans le panneau latéral → vous confirmez chaque étape (ou tout d'un coup) → create_client → create_animal → create_seance → confirmation avec liens vers les 3 fiches.

Choix du modèle d'IA V2

Eliot V2 vous laissera choisir entre deux modèles Anthropic selon vos besoins et votre budget. Configurable depuis Paramètres → Assistant IA.

🚀 Claude Sonnet 4 — Recommandé par défaut
Rapide et économique. Convient à environ 80 % des cas d'usage : recherche d'informations, création de devis et factures, questions analytiques simples.
Coût indicatif : environ 3 USD par million de tokens en entrée, 15 USD par million de tokens en sortie.
Performance : excellent rapport qualité/prix pour l'usage quotidien d'un ERP métier.
⭐ Claude Opus 4 — Raisonnement avancé
Recommandé si vous utilisez Eliot pour des analyses complexes multi-tables, des synthèses business approfondies, ou si Sonnet vous semble approximatif sur vos requêtes.
Coût indicatif : environ 15 USD par million de tokens en entrée, 75 USD par million de tokens en sortie (environ 5× le coût de Sonnet).
Performance : raisonnement multi-étapes plus fiable, meilleures requêtes SQL complexes.
💰
Tarification Anthropic : chaque tenant utilise sa propre clé API Anthropic, achetée directement sur console.anthropic.com. Le solde restant sera affiché en temps réel dans la page Assistant IA. Cap mensuel optionnel à venir.

Limites strictes de sécurité

Ce qu'Eliot ne fait jamais, quelle que soit la version (V1 ou V2). Limites garanties par l'architecture du logiciel, pas seulement par la « bonne volonté » de l'IA.

🚫 Suppression de données
Aucun outil de suppression n'est exposé à Eliot. Clients, animaux, factures, séances : intactes.
🚫 Modification du code
Eliot ne peut pas modifier les pages, le schéma BDD ou la configuration du logiciel.
🚫 Accès aux secrets
Mots de passe, clés API, tokens portail : chiffrés AES-GCM, jamais lus par Eliot.
🚫 Sortie du tenant
Isolation stricte multi-tenant : Eliot d'un compte ne voit jamais les données d'un autre.
🚫 Contournement NF525
Numérotation séquentielle des factures, inaltérabilité : Eliot respecte ces règles.
🚫 Modification audit
Eliot ne peut pas effacer ses propres traces : table d'audit en GRANT INSERT only.
🚫 Registres réglementaires
Registre carnivores, sanitaire : lecture uniquement. Modifications via interface humaine obligatoire.
🚫 Élévation privilèges
Eliot V2 hérite strictement du rôle de l'utilisateur connecté. Pas de bypass possible.
