🚨 Reprise après incident

Diagnostic, redémarrage et restauration — procédures pas-à-pas pour résoudre les incidents en autonomie

🚨En cas d'incident majeur, gardez votre calme. La plupart des problèmes se résolvent en quelques minutes. Ne relancez jamais les containers en boucle — attendez et lisez les logs.
Diagnostic rapide — Identifier le problème
SymptômeCause probableSolution rapide
Page blanche ou erreur 502Container frontend ou nginx arrêtédocker compose restart nginx frontend
API ne répond pas (erreur 500)Container API plantédocker compose restart api puis vérifier les logs
Base de données inaccessibleMariaDB sur MariaDB (LAN) non joignableVérifier que la machine serveur BDD est allumée et accessible
PDF ne génère pasPuppeteer / Chromium dans le container APIRebuild API --no-cache ou redémarrer le container API
Emails ne partent pasConfiguration SMTP incorrecte ou serveur mail downParamètres → SMTP → Tester la configuration
GED inaccessibleDossier C:\AnimGest_GED\ non monté ou suppriméVérifier que le dossier existe et que le volume Docker est monté
Le logiciel est très lentRessources serveur saturéesVérifier RAM et CPU du serveur, fermer les applications inutiles
