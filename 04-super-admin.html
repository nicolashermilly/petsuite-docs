Créer un nouveau contrat
1
Saisie des informations société
Raison sociale, SIRET, adresse, email admin, téléphone. Ces données alimentent la BDD plateforme.
2
Sélection du plan
Socle obligatoire + modules optionnels (CRM, GED étendue, multi-utilisateurs…). Le plan détermine les quotas et accès.
3
Mandat SEPA
Saisie IBAN + BIC. Génération de la référence mandat unique. Engagement minimum 6 mois automatiquement calculé.
4
Provisionnement automatique
Création du schema MariaDB, du bucket S3 GED, de l'URL sous-domaine et du compte administrateur société. Email de bienvenue envoyé.

Provisionnement d'un nouveau tenant
02
Séquence de création
1
Génération du tenant_id
Identifiant unique séquentiel (ex : tenant_0042). Enregistré dans le shard registry de la BDD plateforme.
2
Création du schema MariaDB
Script de provisionnement : CREATE DATABASE anim_gest_tenant_0042. Migration Flyway appliquée automatiquement.
3
Création du bucket S3 GED
Bucket dédié : Anim'Gest-ged-tenant-0042. Politique de cycle de vie hot/warm/cold configurée.
4
Configuration DNS
Sous-domaine : nom-societe.Anim'Gest.fr → pointant vers le reverse proxy. Certificat SSL auto-généré (Let's Encrypt).
5
Création compte admin société
Login = email admin. Rôle ADMIN_SOCIETE. Email de bienvenue avec lien de première connexion (48h).
Commandes de provisionnement
# Créer un nouveau tenant php artisan tenant:create \ --id=tenant_0042 \ --siret=12345678901234 \ [email protected] \ --plan=STANDARD # Vérifier le tenant php artisan tenant:status tenant_0042 # Appliquer migrations php artisan tenant:migrate tenant_0042 # Suspendre accès php artisan tenant:suspend tenant_0042 # Archiver (post-résiliation) php artisan tenant:archive tenant_0042 # Purge définitive (J+5 ans) php artisan tenant:purge tenant_0042 --confirm
La commande purge est irréversible. Vérifiez la date légale avant exécution.

Patches actifs API — découvertes critiques S24
04b
Découverte critique S24 — Injection routes NestJS : _h56.get/post/put() → 404 systématique (filtre NestJS intercepte en amont). SOLUTION : _h56.use() s'exécute AVANT le filtre 404.
⚠️
Règles injection routes
Règle 13 ★Routes injectées → _h56.use() uniquement + préfixe /ged-*
Règle 14 ★Nginx strip /api/ → routes internes SANS ce préfixe dans _h56.use()
BDD injectéemysql2/promise via _q() — jamais app.get(DbService)
ConventionPréfixe /ged-* pour éviter conflits NestJS
