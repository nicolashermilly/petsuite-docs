Règle Docker/OneDrive — Apprentissage S32
⚠️ Avant chaque docker compose build frontend
Exécuter COPIER_TOUTES_PAGES_v2.ps1
Le script est à la racine du projet. Il hydrate les fichiers OneDrive (ReadAllBytes) ET touche leurs timestamps (LastWriteTime = now). Sans ce script, Docker ne transfère que ~28kB de contexte au lieu de ~1MB car les fichiers OneDrive non matérialisés sont ignorés.
Signe d'alerte : "transferring context: 28kB"
Si le log Docker affiche un contexte < 100kB, le script n'a pas été lancé ou les timestamps n'ont pas été touchés. Arrêter le build et relancer COPIER_TOUTES_PAGES_v2.ps1 d'abord.
Contexte correct : "transferring context: ~1MB"
Le COPY . . dans le Dockerfile transfère bien tous les 101 fichiers src/ + public/. Le build inclut les dernières modifications.
