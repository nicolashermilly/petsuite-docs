⚡ Routes API
P0
GET /api/seances/edc — Etudes de Comportement
Route dédiée EDC avec filtre STY_CODE = 'EDC' garanti. Jointures : client, animal, race, statut, praticien. Paramètres : ?statut=, ?client=, ?limit=. Retourne 9 EDC en prod.
GET /api/seances/edc
API
GET/POST /api/leads
GET liste les leads CRM avec statut et source. POST crée un lead depuis le formulaire public (statut NOUVEAU, source FORMULAIRE_WEB). 13 leads en prod.
GET /api/leadsPOST /api/leads
API
GET /api/formulaire/:token
Charge la configuration du formulaire public depuis param_formulaire_lead via token. Retourne 404 si token invalide ou formulaire désactivé. Compatible avec la page /formulaire/[token] existante.
GET /api/formulaire/:token

🎨 Pages frontend
UX
/app/seances/edc — Liste EDC dédiée
Filtre par statut (Planifiées / Réalisées / Annulées), recherche client/animal/puce, stats rapides (total / planifiées / réalisées / sans devis), lien vers fiche séance et devis associé.
UX
/app/elevage/geniteurs
Grille cards avec filtre ♂/♀, âge calculé, nb saillies/portées, résultats santé, titres. Lien vers fiche animal. Utilise v_geniteurs_liste si disponible.
UX
/app/elevage/genealogie
Arbre par portée : affiche mère + père avec race, puce, lien fiche animal. Date saillie si père inconnu. Jointure via elevage_gestation.
UX
/app/elevage/reservations + /app/elevage/reglements
Tableau réservations avec total acomptes, filtre client/portée, statut badge. Page réglements : récap 3 compteurs + tableau des acomptes versés.

🔧 Corrections techniques
FIX
custom-routes.ts — suppression code orphelin
Le script Python de suppression des blocs v1 laissait du code tronqué (if/finally hors contexte). Correction via comptage d'accolades ouvrantes/fermantes. Script repair_and_rebuild_s46.ps1.
FIX
Scripts PS — suppression des && et caractères spéciaux
PowerShell 5 ne supporte pas && comme séparateur. Tous les scripts utilisent maintenant des commandes séparées. Les strings PS n'utilisent plus de caractères accentués.
FIX
Routes élevage — noms colonnes corrigés
v1 utilisait POR_MERE_ID et SAI_ID (inexistants). Colonnes réelles confirmées via diag : ANI_ID_FEMELLE, GES_ID, jointure via elevage_gestation. Routes v2 auto-adaptatives via information_schema.
