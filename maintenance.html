1. Architecture actuelle

Le site no-sages-editor.com est un site 100 % statique hébergé par GitHub Pages, avec le domaine géré chez Ionos.

Chaîne complète
  1. Ionos (domaine + DNS + emails) — résolution du nom de domaine et gestion des enregistrements mail.
  2. DNS — 4 enregistrements A vers les IP GitHub Pages + 1 CNAME www vers nicolashermilly.github.io.
  3. GitHub Pages — sert automatiquement le contenu de la branche main du repo nicolashermilly/AnimGest-docs.
  4. Let's Encrypt — certificat HTTPS géré automatiquement par GitHub Pages, renouvellement transparent.

2. Mettre à jour le site

Le workflow est le même que pour n'importe quelle page du repo.

Procédure standard 
cd C:\AnimGest-docs
git pull --rebase
# ... modifier les fichiers ...
git add .
git commit -m "Description de la modification"
git push

Le site est mis à jour automatiquement sur https://no-sages-editor.com en moins d'une minute.

Fichier CNAME à la racine

Le fichier CNAME (sans extension) à la racine du repo doit contenir exactement no-sages-editor.com, sans https:// et sans retour à la ligne. Ne jamais le supprimer : GitHub Pages en a besoin pour servir le domaine personnalisé.

3. Vérifier le bon fonctionnement

Test manuel rapide

Ouvrir dans un navigateur : https://no-sages-editor.com. Vérifier :

  • La page s'affiche avec le design Anim'Gest (pas une page d'erreur GitHub ni le 404 par défaut).
  • Le cadenas 🔒 HTTPS est présent dans la barre d'adresse.
  • Le certificat est valide (cliquer sur le cadenas → voir Émis par Let's Encrypt).

Vérification DNS en ligne de commande

Resolve-DnsName no-sages-editor.com -Type A
Resolve-DnsName www.no-sages-editor.com -Type CNAME

Résultats attendus : les 4 IP 185.199.108-111.153 pour l'apex, et nicolashermilly.github.io pour www.

Vérification côté GitHub Pages

Aller sur github.com/nicolashermilly/AnimGest-docs/settings/pages. Vérifier :

  • Vérification DNS réussie (message vert sous le champ Custom domain).
  • Imposer le protocole HTTPS : case cochée.
  • Dernier déploiement récent (le commit en cours).

4. Incidents fréquents

Le site affiche un 404 GitHub alors qu'il marchait avant

Le fichier CNAME à la racine du repo a probablement été supprimé accidentellement. Le recréer avec comme contenu no-sages-editor.com (sans BOM, sans retour à la ligne), committer et pusher.

Le cadenas HTTPS n'est plus valide

Cela peut arriver après une modification DNS. Aller dans Settings → Pages du repo, décocher puis recocher Imposer HTTPS. Si le certificat ne se régénère pas, retirer et remettre le Custom domain.

Une page modifiée n'apparaît pas en ligne

Vérifier que le commit a bien été pushé (git log origin/main --oneline -5). GitHub Pages met environ 1 minute à déployer. Vider le cache du navigateur (Ctrl+Shift+R) si le contenu ancien reste affiché.

Les emails @no-sages-editor.com ne fonctionnent plus

Vérifier dans la zone DNS Ionos que les enregistrements MX, TXT (SPF, DMARC) et CNAME (DKIM, autodiscover) service Mail sont toujours présents. Ils ne doivent jamais être supprimés.

5. Ce qu'il ne faut jamais faire

Ne jamais toucher aux enregistrements Mail côté Ionos

Les enregistrements MX, SPF, DMARC, DKIM et autodiscover doivent rester intacts, même si aucune adresse email n'est actuellement créée. Les supprimer empêche la création future d'adresses pro.

Ne jamais supprimer le fichier CNAME

Sans ce fichier à la racine du repo, GitHub Pages bascule sur le domaine par défaut nicolashermilly.github.io et le site no-sages-editor.com renvoie une erreur.

Ne jamais pusher d'informations sensibles dans ce repo public

Les credentials, clés API, secrets SMTP et documents confidentiels doivent rester dans le repo privé AnimGest-docs-private. Une fausse protection JS côté client n'a aucune valeur de sécurité.

Ne jamais force-push sur main sans backup préalable

Toute opération destructive (git filter-repo, git reset --hard, git push --force) doit être précédée d'un backup complet du repo .git inclus, dans C:\Backup\.

Ne pas modifier la branche source de GitHub Pages

La branche servie par GitHub Pages est main à la racine. Ne pas la changer sauf raison explicite : cela casse immédiatement le site.
