Brancher Claude Desktop sur Anim'Gest

Interagir avec votre cabinet en langage naturel via le serveur MCP Anim'Gest. Beta technique

1. À quoi ça sert

Le serveur MCP Anim'Gest (Model Context Protocol) permet à un client compatible — typiquement Claude Desktop — d'accéder à vos données métier via des commandes en langage naturel. Plutôt que cliquer dans l'interface, vous pouvez demander :

  • "Crée-moi un devis pour le client Dupont, séance d'1 heure à 80€ TTC."
  • "Quels clients ont une consultation prévue cette semaine ?"
  • "Liste les factures impayées de plus de 30 jours."

Toutes les actions passent par les mêmes règles métier que l'interface web : validations, archivage, déclenchement automatique des règles internes. Aucune donnée n'est altérée en contournant Anim'Gest.

3. Configuration de Claude Desktop

3.1 Localisation du fichier de configuration

SystèmeChemin
Windows%APPDATA%\Claude\claude_desktop_config.json
macOS~/Library/Application Support/Claude/claude_desktop_config.json
Linux~/.config/Claude/claude_desktop_config.json

Si le fichier n'existe pas, créez-le.

3.2 Configuration recommandée — mode local Nouveau

Le mode recommandé exécute le serveur MCP localement comme sous-processus de Claude Desktop. Pas de port HTTP à configurer, pas de proxy intermédiaire, pas de connexion réseau exposée. 

{
  "mcpServers": {
    "animgest": {
      "command": "node",
      "args": [
        "<CHEMIN_FOURNI_PAR_VOTRE_ADMINISTRATEUR>"
      ],
      "env": {
        "ANIMGEST_TOKEN": "<TOKEN_FOURNI_PAR_VOTRE_ADMINISTRATEUR>"
      }
    }
  }
}

Votre administrateur Anim'Gest vous fournit le chemin exact (selon votre OS) et le token personnel. Aucune autre information n'est nécessaire : la configuration locale du cabinet est gérée côté serveur Anim'Gest.

3.3 Configuration alternative — mode distant

Si votre cabinet utilise une instance MCP hébergée (déploiement multi-cabinets), votre administrateur peut vous fournir une configuration distante via SSE. Format générique : 

{
  "mcpServers": {
    "animgest": {
      "command": "<COMMANDE_FOURNIE>",
      "args": [ "<ARGUMENTS_FOURNIS>" ],
      "env": {
        "ANIMGEST_TOKEN": "<TOKEN>"
      }
    }
  }
}

3.4 Redémarrage

Après modification du fichier :

  1. Fermez complètement Claude Desktop (vérifiez la barre des tâches)
  2. Relancez l'application
  3. Allez dans Paramètres › Intégrations — vous devez voir animgest avec une pastille verte

Si la pastille est rouge ou orange, voir la section Dépannage.

5. Exemples d'utilisation

Exemple 1 — Recherche client

Vous : Trouve-moi le client Dupont.

Claude Desktop : J'ai trouvé 2 clients correspondants :

  • Dupont Martin — martin.dupont@example.com — 06 12 34 56 78
  • Dupont SAS — contact@dupont-sas.com — 04 56 78 90 12

Exemple 2 — Statut du cabinet

Vous : Quel est mon abonnement Anim'Gest ?

Claude Desktop : Votre cabinet est sur la formule Pro, active jusqu'au 31/12/2027. Votre dernière facture (avril 2026, 49€ HT) est payée.

Exemple 3 — Création de devis

Vous : Crée un devis pour Dupont Martin : 1 séance de bilan comportemental à 80€ HT, TVA 20%.

Claude Desktop : Devis DEV-2026-0123 créé pour Dupont Martin. Total HT 80€, TVA 16€, Total TTC 96€. Vous pouvez le valider et l'envoyer depuis Anim'Gest.

6. Sécurité

  • Votre token est personnel. Ne le partagez avec personne, ni par email, ni dans un message Slack, ni dans un dépôt Git.
  • Le token expire automatiquement au bout d'1 heure. Régénérez-le auprès de votre administrateur si besoin.
  • Toutes les actions sont tracées dans le journal d'audit Anim'Gest (qui, quoi, quand) — RGPD conforme.
  • Aucune donnée d'autre cabinet n'est accessible. Votre token est lié à votre cabinet uniquement.
  • Aucune suppression possible. Les agents IA n'ont jamais le droit de supprimer des données dans Anim'Gest, peu importe la formulation de la demande.

7. Dépannage

SymptômeSolution
Pastille rouge "Failed to spawn process" Le chemin du serveur MCP fourni est invalide ou le fichier n'a pas été déployé. Contactez votre administrateur.
Pastille rouge "Token manquant" Le token n'est pas présent dans la configuration. Vérifiez la valeur de ANIMGEST_TOKEN.
Pastille rouge "Token invalide" Votre token est expiré ou malformé. Demandez-en un nouveau à votre administrateur.
Pastille rouge "Cabinet inactif" L'abonnement Anim'Gest du cabinet n'est pas actif. Contactez l'administrateur.
Pastille orange "Formule requise" L'action demandée nécessite la formule Pro ou Expert. Voir les conditions d'abonnement.
Claude Desktop semble bloqué sur "Initializing..." Patientez 30 secondes puis redémarrez Claude Desktop. Si le problème persiste, contactez le support.

8. Limitations beta

  • Mode beta technique — disponibilité conditionnée à la livraison infrastructure multi-tenant.
  • 4 capacités exposées en V0, l'ensemble des fonctionnalités Anim'Gest sera couvert progressivement en V1.
  • Une seule session Claude Desktop par token recommandée.

9. Support

Pour toute question, demande de token, ou bug rencontré : contactez votre administrateur Anim'Gest ou écrivez à support@animgest.fr.
