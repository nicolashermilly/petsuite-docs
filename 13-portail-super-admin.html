01Accès & sécurité
🔐
Compte super admin unique
LoginSuperAdmin — identifiant unique, non modifiable
URL portailapp.Anim'Gest.fr/superadmin — route distincte des tenants
MFATOTP obligatoire (Google Authenticator / Authy)
SessionJWT 15 min + refresh 2h — expiration courte intentionnelle
IP whitelistOptionnel : restreindre aux IPs connues (bureau, VPN)
AuditToutes les actions loggées dans audit_plateforme avec IP + timestamp
⚠️
Ce que le super admin peut faire
Voir toutes sociétésLecture de tous les tenants, contrats, prélèvements
Créer un tenantProvisionnement manuel ou automatique
Suspendre / RésilierBloquer l'accès d'une société
Gérer les CGVPublier une nouvelle version avec notification
Accéder aux logsJournal d'audit complet toutes sociétés
Déclencher backupsBackup manuel ou restauration d'urgence
Le super admin ne peut jamais accéder aux données métier d'un tenant (animaux, factures, séances). Seules les données de pilotage sont accessibles.

Workflow gestion impayés
1
Webhook GoCardless reçu (payment.failed)
Mise à jour automatique prelevement.pre_statut = ECHOUE. Email envoyé au client. Compteur tentative incrémenté.
2
Alerte super admin
Notification email + Slack avec nom tenant, montant, numéro de tentative. Visible dans la section "Alertes actives" du dashboard.
3
2e tentative automatique (J+8)
Si pre_tentative_num = 1 et toujours impayé après 7 jours : nouvelle tentative soumise à GoCardless automatiquement.
4
Suspension si 2e échec
ten_statut → SUSPENDU. L'admin super admin peut manuellement contacter le client, accepter un paiement virement, ou résilier.
5
Réactivation manuelle
Depuis la fiche tenant : bouton "Réactiver" disponible après règlement vérifié. Nouveau mandat SEPA si nécessaire.

Processus de mise à jour
1
Rédiger nouvelle version
Saisir le texte complet dans l'éditeur. Hash SHA-256 calculé automatiquement. Statut = DRAFT.
2
Définir date de mise en vigueur
Minimum J+30 depuis aujourd'hui. Notifications clients planifiées automatiquement.
3
Envoyer les notifications
Email automatique à tous les admins de tenants ACTIF : "Nos CGV évolueront le [date]."
4
Publication automatique à J
cgv_actuelle basculée. Tous les clients devront re-accepter à leur prochaine connexion.
