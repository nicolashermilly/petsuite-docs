Eliot V2 — Refonte agent IA prévue Roadmap S122

La mémoire persistée d'Eliot (présentée ici, livrée S69) reste pleinement fonctionnelle en V2. Eliot V2 ajoutera 2 nouvelles tables compagnons : eliot_audit (traçabilité complète des actions) et eliot_permissions (config niveaux d'autorisation par tool).

Voir section « Évolution Eliot V2 » plus bas pour le détail des changements et complémentarités entre mémoire (S69) et audit (S122).

🧠

Vue d'ensemble

Avant la mémoire persistée (livrée S69), Eliot repartait de zéro à chaque session. La mémoire lui permet de se souvenir des préférences du praticien, du contexte métier, et des informations importantes sur les animaux et clients — sans que le praticien ait à se répéter.

💾
Faits persistants
Table eliot_memoire — faits catégorisés (PREFERENCE, CONTEXTE, ANIMAL, CLIENT, PROCEDURE) avec niveau de confiance et expiration optionnelle.
📖
Historique résumé
Table eliot_contexte_session — résumé automatique de chaque conversation pour garder une trace légère de l'historique.
🔧
Tool memoriser
Eliot peut lui-même décider de mémoriser un fait important quand le praticien mentionne une information récurrente.
👁️
Page admin
Page /app/parametres/eliot-memoire pour visualiser, corriger et supprimer les faits mémorisés par Eliot.
🗃️

Injection dans le prompt

À chaque appel POST /ia/chat, la mémoire est lue depuis la BDD et injectée en fin du system prompt, sous forme de section ## Mémoire contextuelle.

💡
Maximum 30 faits injectés (triés par importance et nombre de mentions). Seuls les faits EM_ACTIVE = 1 sont inclus. Si la mémoire est vide, rien n'est ajouté — comportement identique à l'origine.
System prompt enrichiPrompt
[Prompt système Eliot existant...]

## Mémoire contextuelle
### PREFERENCE
- langue_reponse: français
- format_dates: JJ/MM/AAAA

### CONTEXTE
- metier_principal: Comportementaliste canin
- heure_ouverture: 9h-18h lundi-samedi

### ANIMAL
- ani_22_allergies: Frimousse allergique au poulet

Tool memoriser

Eliot appelle ce tool quand il détecte une information récurrente ou une préférence exprimée par le praticien.

ParamètreTypeDescription
categorieenumPREFERENCE · CONTEXTE · ANIMAL · CLIENT · PROCEDURE · ALERTE
clestringIdentifiant court — ex: ani_22_allergies
valeurstringValeur du fait à mémoriser
importancenumber1-10 : priorité d'injection (défaut 5)
💡
Eliot utilise une importance élevée (7-10) pour les informations stratégiques (allergies, préférences confirmées) et une importance basse (3-5) pour les déductions ou informations contextuelles.
🖥️

Page admin

Page /app/parametres/eliot-memoire permettant au praticien de contrôler ce qu'Eliot a mémorisé.

  • 👁️
    Tableau filtrable par catégorie : Catégorie · Clé · Valeur · Importance · Date MàJ
  • 🗑️
    Bouton « Oublier » par ligne — le praticien peut corriger les faits incorrects
  • 📖
    Section historique — 20 dernières sessions avec résumé automatique et nombre de messages
  • ⚠️
    Bouton « Vider toute la mémoire » avec confirmation — pour repartir de zéro

Évolution Eliot V2 — Audit complémentaire

Eliot V2 (S122) introduira 2 nouvelles tables compagnons à la mémoire persistée, qui couvriront un besoin différent : la traçabilité des actions et la gestion des permissions. La mémoire reste sur le « savoir » (faits), l'audit couvre le « faire » (actions).

🧠
eliot_memoire S69 LIVRÉ
Ce que sait Eliot : préférences, contexte métier, informations animaux/clients récurrentes. Faits persistants entre les sessions. Inchangé en V2.
📊
eliot_audit V2 S122
Ce que fait Eliot : trace chaque action (lecture, préparation, confirmation, exécution, refus). Qui, quand, quel outil, quel résultat. ~22 colonnes pour une visibilité complète.
🔐
eliot_permissions V2 S122
Ce que peut faire Eliot : configuration par tool des niveaux d'autorisation (AUTO / CONFIRM / DENIED) et rôles minimums requis (ADMIN / MANAGER / ADMINISTRATIF).

Complémentarité mémoire ↔ audit

La mémoire et l'audit servent des cas d'usage différents et sont totalement complémentaires :

AspectMémoire (S69)Audit V2 (S122)
Granularité Faits durables (mois/années) Action ponctuelle (timestamp précis)
Modifiable Oui (oublier, corriger via UI) Non — INSERT only, intégrité garantie
Rôle dans Eliot Injecté dans system prompt pour contextualiser Trace post-hoc pour conformité et débogage
Volumétrie Stable (~50-200 faits par tenant) Croissante (~50-100 entrées/jour si usage actif)
Rétention Permanent (sauf oubli explicite) 12 mois par défaut, configurable
Page admin /parametres/eliot-memoire (existante) /parametres/eliot-audit (nouvelle V2)

Impact V2 sur la mémoire existante

La mémoire persistée S69 n'évolue pas structurellement en V2. Les changements concernent uniquement les nouvelles couches :

  • ✅
    La table eliot_memoire reste identique, aucune migration de données nécessaire
  • ✅
    Le tool memoriser reste en niveau AUTO (pas de workflow CONFIRM nécessaire — il s'agit de simple mémorisation)
  • ✅
    La page admin /parametres/eliot-memoire reste accessible et fonctionnelle
  • 🆕
    Chaque appel au tool memoriser sera tracé dans eliot_audit (statut READ, action MEMORISATION)
  • 🆕
    Le tool memoriser respectera les permissions héritées : tous les rôles peuvent l'utiliser par défaut
  • 🆕
    Possibilité de désactiver memoriser via eliot_permissions.EP_LEVEL = 'DENIED' par tenant qui ne souhaite pas la mémorisation automatique
✅
Aucune action requise pour les utilisateurs actuels : la mémoire continue de fonctionner pendant et après la migration V2. Les nouvelles fonctionnalités d'audit et permissions seront livrées comme des extensions, pas comme des remplacements.
📅
Calendrier : refonte Eliot V2 prévue S122 (~4 jours développement, vertical slice métier comportementaliste). Les 2 nouvelles tables seront créées dans toutes les BDD tenant en J1 de la livraison.
