Structure PATCH_ROUTES_S56
await app.listen(port, "0.0.0.0"); // PATCH_ROUTES_S56 — toutes les routes injectées ICI try { const _h56 = app.getHttpAdapter().getInstance(); const _db56 = app.get(require('./common/db.service').DbService); _h56.put('/seances/types/:id', ...) _h56.use('/ged-libelles-ec', ...) // S24 — GET/PUT 64 libellés EC _h56.use('/ged-edc', ...) // S25 — CRUD fiche EDC _h56.use('/ged-cac', ...) // S25 — CAC par séance + initial _h56.use('/ged-evolution-cac', ...) // S25 — Historique CAC animal // Chargé via : const _injectEdc = require('./patch_edc_cac_s25'); _injectEdc(_h56); } catch(_e56) { console.warn('PATCH_ROUTES_S56 error:', _e56.message); }
Les routes DOIVENT être dans le bloc try { const _h56 = ... } — jamais entre await app.listen et ce bloc.
Découverte critique S24 — Injection routes NestJS
_h56.get/post/put() → 404 systématique — le filtre NestJS intercepte avant d'atteindre la route injectée.
🔴
Problème identifié
_h56.get/post/put()404 systématique — filtre NestJS intercepte en amont
app.get(DbService)Ne fonctionne pas dans ce contexte — retourne undefined
✅
Solution validée
_h56.use()S'exécute AVANT le filtre 404 NestJS — fonctionne
mysql2/promiseAccès BDD direct via _q() — pas app.get(DbService)
Préfixe /ged-*Convention pour éviter les conflits NestJS
Nginx strip /api/Routes internes enregistrées SANS le préfixe /api/
Pattern correct pour routes injectées S24+
// ✅ CORRECT — _h56.use() s'exécute avant le filtre 404 NestJS // Nginx strip /api/ → routes internes enregistrées SANS /api/ _h56.use('/ged-libelles-ec', async (req, res) => { // BDD : mysql2/promise directement via _q(), pas app.get(DbService) const rows = await _q('SELECT * FROM param_libelles_ec'); res.json(rows); }); // Routes actives S25 (préfixe /ged-* par convention) _h56.use('/ged-libelles-ec', ...) // → GET/PUT /api/ged-libelles-ec _h56.use('/ged-edc', ...) // → GET/POST/PUT /api/ged-edc/* + problèmes _h56.use('/ged-cac', ...) // → GET/POST /api/ged-cac/:id + initial _h56.use('/ged-evolution-cac', ...) // → GET /api/ged-evolution-cac/:ani_id // ❌ NE PLUS UTILISER pour nouvelles routes // _h56.get('/edc', ...) → 404 systématique
