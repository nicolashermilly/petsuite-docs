02 Règles absolues — NE JAMAIS enfreindre
🚫 custom-routes.ts — app.use() uniquement
NestJS intercepte app.get() / app.post() / app.put() — utiliser exclusivement app.use('/route', async (req, res) => {}). Toutes les routes dans registerCustomRoutes(app).
🚫 Jamais de backticks dans les strings SQL TypeScript
Les template literals avec ${} dans les strings SQL causent des erreurs de compilation. Utiliser la concaténation ou les double quotes. Exception légitime : interpolation explicite de valeurs non-paramétrées.
🚫 PDF — jamais window.open() ni navigation page
Tous les PDF doivent s'ouvrir dans une Modal/Popup avec srcdoc ou iframe. Jamais de nouvelle page ou onglet.
🚫 CAC — jamais de lien sidebar
Le suivi CAC est accessible UNIQUEMENT depuis /app/seances/[id] et /app/seances/[id]/etude-comportementale. Aucun lien direct dans la sidebar.
⚠️ Docker — toujours cd deploy avant docker compose
cd "...\micro_logiciel_frontend_nextjs\deploy" obligatoire avant toute commande docker compose. Rebuild API : build --no-cache → up -d → restart nginx. Jamais restart seul pour l'API.
⚠️ Frontend — COPIER_TOUTES_PAGES_v2.ps1 avant tout build
Depuis la racine frontend : .\COPIER_TOUTES_PAGES_v2.ps1 obligatoire avant docker compose build frontend. Vérifier "transferring context: ~1MB" (28kB = script non lancé).
⚠️ Scripts Python — jamais dans OneDrive
Écrire dans $env:USERPROFILE\Downloads\ uniquement. Le dossier OneDrive cause [Errno 13] Permission denied.
⚠️ TSX — "use client" ligne 1 absolue
"use client" doit être la première ligne de tout fichier .tsx de page. Pas d'apostrophes françaises dans les strings TypeScript.
ℹ️ Git petsuite-docs — git pull --rebase avant push
Le remote avance fréquemment. Toujours git pull --rebase avant git push pour éviter les conflits.
ℹ️ PowerShell — -LiteralPath pour les chemins [id]
Les chemins contenant [id] nécessitent -LiteralPath en PowerShell. Les chemins avec [ et ] sont interprétés comme des wildcards.
03 Pattern custom-routes.ts
ℹ️ Fichier source : api/src/custom-routes.ts — ~2800 lignes — 90+ routes — 87+ appels getDb() — 0 createConnection direct
// Pattern obligatoire — route avec body parsing manuel
app.use("/ma-route", async (req: any, res: any) => {
  // 1. Body parsing manuel (obligatoire pour POST/PATCH)
  const b: any = await new Promise<any>((resolve) => {
    if (req.body && Object.keys(req.body).length > 0) { resolve(req.body); return; }
    let data = "";
    req.on("data", (chunk: any) => { data += chunk; });
    req.on("end", () => { try { resolve(JSON.parse(data)); } catch { resolve({}); } });
  });

  // 2. Connexion BDD — toujours getDb()
  const db = await getDb();
  try {
    const [rows]: any = await db.execute("SELECT * FROM ma_table WHERE ID = ?", [b.id]);
    res.json(rows);
  } catch (e: any) {
    res.status(500).json({ error: e.message });
  } finally {
    await db.end(); // TOUJOURS fermer
  }
});
